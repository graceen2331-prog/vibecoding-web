# Vibe Coding 产品规划文档 (MVP)

## 1. 产品概述

### 产品名称
Vibe Coding - 初学者编程学习平台

### 产品使命
为零基础编程新手提供**低门槛、高参与度、充满正向反馈**的编程学习体验。通过"Vibe"（氛围感）设计和即时反馈机制，让初学者30秒内就能运行第一行代码，建立学习自信心。

### 目标用户
- **主要用户**：15-35岁编程初学者（完全零基础）
- **学习动机**：了解编程、培养逻辑思维、获得成就感
- **关键特征**：需要友好的学习环境、即时反馈、看得见的进度

## 2. MVP 范围与核心特性

### 2.1 学习内容 - 阶段一课程（JavaScript）

#### 课程结构 (8 节核心课程)

| 课程号 | 课程名称 | 学习时长 | 核心概念 | 可视化项目 |
|-------|---------|---------|---------|----------|
| 课程 1 | "你好，JavaScript" | 15分钟 | 变量、输出 | 显示欢迎信息 |
| 课程 2 | "数字的世界" | 20分钟 | 数据类型、数值运算 | 简单计算器 |
| 课程 3 | "判断与选择" | 20分钟 | if/else 条件判断 | 成绩评级系统 |
| 课程 4 | "重复的力量" | 25分钟 | for 循环基础 | 绘制图案或计数 |
| 课程 5 | "函数的魔力" | 25分钟 | 函数定义与调用 | 实现自己的功能 |
| 课程 6 | "列表与集合" | 25分钟 | 数组基础、迭代 | 购物清单应用 |
| 课程 7 | "字符串变魔术" | 20分钟 | 字符串操作 | 文本转换工具 |
| 课程 8 | "综合项目：互动小作品" | 60分钟 | 综合应用前7课概念 | **可分享的交互应用** |

**课程 8 项目示例**：学生可选择以下之一完成
- 🎮 **交互式选择冒险游戏** - 基于用户输入讲述故事
- 📊 **个性化成绩分析工具** - 输入分数自动分析和建议
- 🎨 **动态艺术生成器** - 通过参数生成不同的图案和颜色

### 2.2 核心功能一：交互式编辑器

#### 技术实现
- **JavaScript 运行环境**：WebContainers (StackBlitz)
- **编辑器组件**：VS Code 风格编辑器（基于 Monaco 或 CodeMirror）
- **实时反馈**：代码更改即时执行，输出实时显示

#### 用户体验要求
- ⚡ **30秒内可运行代码**：打开课程→按下"运行" → 看到结果
- 🎯 **引导式代码框架**：每课有预填充的代码框架，学生填空即可运行
- 💡 **实时错误提示**：语法错误实时显示，鼓励性语言提示

## 3. 技术架构

### 3.1 技术栈选型

| 层级 | 技术 | 原因 |
|-----|------|------|
| **前端** | React + Vite + Tailwind CSS | 快速、现代、易于扩展 |
| **后端** | Node.js (Express) | 快速迭代 MVP，与前端技术栈统一 |
| **代码运行** | WebContainers (JS) | 无需本地环境，浏览器内运行 |
| **数据库** | PostgreSQL + Supabase | 无服务架构，快速部署 |
| **认证** | Magic Link + OAuth (GitHub) | 低摩擦登录，符合初学者 |
| **分析** | PostHog | 追踪用户学习路径，优化课程 |

### 3.3 API 端点规划（MVP）

```
# 认证
POST /api/auth/login         - Magic Link 登录
POST /api/auth/verify        - 验证 Magic Link token

# 用户
GET  /api/user/profile       - 获取用户信息
GET  /api/user/progress      - 获取学习进度

# 课程
GET  /api/courses            - 获取所有课程列表
GET  /api/courses/:id        - 获取单个课程详情
POST /api/courses/:id/complete - 标记课程完成

# 成就
GET  /api/achievements       - 获取用户成就列表
POST /api/achievements/check - 检查是否解锁新成就

# 项目分享
POST /api/projects           - 创建项目分享
GET  /api/projects/:id       - 获取项目详情
GET  /api/projects/:id/share - 获取分享链接

# 代码验证
POST /api/code/validate      - 验证用户代码是否正确
```

## 4. 用户流程 (User Flow)

### 4.1 首次用户流程

```
1. 访问网站 → 看到"30秒学会编程"的宣传语
   ↓
2. 点击"现在开始" → Magic Link 登录
   ↓
3. 进入"课程 1：你好，JavaScript"
   ↓
4. 看到讲解 + 预填充代码
   ↓
5. 点击"运行" → 即时看到输出 (欢迎信息)
   ↓
6. ✅ 任务完成！获得第一个徽章 "第一行代码"
   ↓
7. 显示："你已解锁新成就！点击继续→课程 2"
```

### 4.2 学习循环流程 (重复 8 次)

```
进入课程 → 阅读讲解 → 观看 2分钟视频 → 填空编码 → 点击运行 
  ↓                                            ↓
  ←←←←← (需要修改?) ←←←←←←←← 错误提示
  
代码正确 → 显示成功动画 → 获得经验值 → 解锁下一课/新徽章
```

### 4.3 第 8 课特殊流程

```
课程 8 讲解 → 选择项目主题 (冒险游戏/成绩工具/艺术生成器) 
  ↓
看示例 + 代码框架 
  ↓
编写完整代码 (不是填空，是完全自主) 
  ↓
点击运行测试 
  ↓
✅ 通过验证 → 获得"全能开发者"徽章 
  ↓
点击"分享我的作品" 
  ↓
获得分享链接 → 分享到微博/微信
```

## 5. 成功指标 (KPI)

### 5.1 MVP 阶段关键指标

| 指标 | 目标 | 衡量方法 |
|-----|------|---------|
| 新用户留存（Day 1） | > 60% | 完成课程1后的返回率 |
| 课程完成率 | > 40% | 完成8课的用户比例 |
| 作品分享率 | > 25% | 课程8完成者中实际分享的比例 |
| 平均学习时长 | 3-5小时 | 从课程1到课程8的总耗时 |
| 用户满意度 | > 4.5/5 | 课程完成后的评分 |

### 5.2 用户体验指标

- 从登录到运行第一行代码的时间：< 30秒
- 代码编辑器响应速度：< 100ms
- 移动端适配度：100%

## 7. 数据模型初步设计

### 用户表 (users)
```sql
id (PK)
email
username
auth_provider (magic_link / github)
created_at
last_login_at
```

### 课程进度表 (user_progress)
```sql
id (PK)
user_id (FK)
course_id (FK)
status (not_started / in_progress / completed)
code_submission (最后提交的代码)
completed_at
```

### 成就表 (achievements)
```sql
id (PK)
user_id (FK)
achievement_type (徽章类型)
unlocked_at
```

### 项目分享表 (shared_projects)
```sql
id (PK)
user_id (FK)
course_id (FK)
project_data (JSON - 项目参数)
share_token (分享链接)
created_at
view_count
```

## 9. 备注

- 所有课程内容均强调"友好、鼓励"的基调
- 错误提示尽量避免"失败"字眼，改为"还差一点"、"再试试"
- 每个成就解锁时，要有清晰的视觉和声音反馈
- 移动优先设计，确保手机上的学习体验
